Single Column Layout Pattern

This page demonstrates a single-column layout where content sections span the full width of the grid at all breakpoints. This pattern is ideal for landing pages, documentation, and content-first experiences where readability and flow are prioritized.

Responsive Typography Rhythm

All typography on this page uses Carbon design tokens to maintain consistent rhythm across breakpoints. Headings scale automatically using Carbon's type styles, and spacing between sections uses Carbon's spacing tokens for predictable vertical flow.

The mobile-first approach means small viewport styles serve as the base, with progressive enhancements at medium and large breakpoints. This ensures optimal readability on all device sizes without custom media queries.

Key Takeaways

Grid Classes: All content sections have grid classes applied directly to the section element (no wrapper divs). This maintains clean DOM structure and prevents grid-breaking nesting.

BEM Naming: Custom classes follow the Block__Element pattern (e.g., .content-section, .content-section__heading) to distinguish them from Carbon's component classes and maintain predictable CSS specificity.

Image Dimensions: All images include explicit width and height attributes to prevent layout shift during page load. This is a constitution requirement for performance and accessibility.
